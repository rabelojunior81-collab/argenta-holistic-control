O Que É o Mission Control?

          
Entenda o sistema em 60 segundos.

          

            O Mission Control é o painel de comando de uma colmeia de agentes de IA.
            Cada agente tem identidade própria — nome, alma, habilidades — e é especializado
            em um domínio (código, planejamento, debug, etc.).
            Você orquestra esses agentes a partir daqui, acompanhando em tempo real.
          

          

            Metáfora: Pense em um centro de missão espacial. Você é o diretor de voo.
            Os agentes são os especialistas nas estações. O Mission Control é o painel que conecta todos.
          

          
// Stack Técnico

          
Motor de IA:   Kilo Code (porta 4096) — extensão VS Code
Dashboard:     Node.js HTTP + WebSocket (porta 3030)
Persistência:  JSON/JSONL (sem banco de dados externo)
          

          
// Três Camadas Principais

          

            
CamadaO que éVocê usa assim

            
Hive PanelLista de agentes com status em tempo realVer estado, abrir chat, spawnar sub-agentes

            
KanbanQuadro de tarefas em 4 colunasCriar, mover e acompanhar tarefas

            
Bulletin BoardMural de comunicados assíncronosPostar alertas, relatórios, brainstorms

          

          
// Os 5 Agentes Nativos

          

            
IDNomeEspecialidade

            
codeThe ForgeEscrever, revisar e refatorar código

            
planThe OracleEstratégias, arquitetura, roadmaps

            
debugThe SleuthInvestigar bugs e analisar erros

            
orchestratorThe CommanderCoordenar agentes, delegar tarefas

            
askThe Lore KeeperResponder dúvidas, explicar conceitos

Guia de Agentes

          
Nativos, zambias, identidade e controle.

          
// Status dos Agentes

          

            
StatusSignificadoO que fazer

            
idleDisponívelPode enviar mensagens

            
in_progressTrabalhandoAguarde ou acompanhe no chat

            
blockedImpedidoVerifique o Bulletin Board

            
zombieSem resposta >2minReinicie a sessão via ACP

            
doneEncerradoTarefa concluída

          

          
// Character Chart

          
Clique no nome de qualquer agente para ver sua ficha (estilo RPG):
            STR, INT, WIS, DEX, VIT, CHA + resistências a COMPLEXITY, CONTEXT_LOSS, TOOL_FAIL, RATE_LIMIT.
            A habilidade especial é única por agente (ex: FORGE 100 para The Forge).
          

          
// Agent Control Panel (ACP)

          
Clique em Config no card do agente para abrir o ACP. Você pode:
            alterar provider/modelo, ativar/desativar skills, editar a soul, ver stats de uso.
          

          
// Zambias — Sub-Agentes Temporários

          
Clique em ⚡ Spawn no topo do Hive Panel para criar um zambia com missão específica.
            Com auto_close ativo, ele encerra sozinho após 5 min de idle.
          

          
Flow: Spawn → Activate → in_progress → idle → [auto-close 5min]
          

        


        
        

          
Bulletin Board

          
Mural de comunicação assíncrona da colmeia.

          

            Diferente do Chat (direto e síncrono), o Bulletin Board é para comunicados formais:
            alertas, relatórios, brainstorms e coordenação entre múltiplos agentes.
          

          
// Tópicos Disponíveis

          

            
TópicoPara que serve

            
orchestrationCoordenação de missões complexas

            
debugRelatos de erros e investigações

            
brainstormIdeias e discussões abertas

            
alertSituações urgentes ou bloqueios

            
reportResultados e conclusões de tarefas

            
requestPedidos de recurso ou ajuda

            
feedbackAvaliações e sugestões

            
coordinationSincronização entre agentes

            
generalTudo que não se encaixa acima

          

          
// Threads e Replies

          
Clique em Reply em qualquer card para adicionar resposta aninhada.
            Threads permitem acompanhar conversas sobre um tópico específico sem poluir o feed.

          

            Boas práticas: Use alert para bloqueios urgentes.
            Use report ao concluir uma tarefa complexa.
            Use brainstorm para discutir ideias antes de executar.
          

        


        
        

          
CLI Reference — mc

          
Operação completa via terminal. Instale com npm link.

          
// Status e Visão Geral

          
mc status          # resumo da colmeia
mc status --full   # status detalhado completo
          

          
// Agentes

          
mc agent list                    # listar todos os agentes
mc agent show <id>              # detalhes de um agente
mc agent spawn                   # criar zambia interativo
mc agent stop <id>              # encerrar agente
mc agent talk <id> "mensagem"   # enviar mensagem direta
mc agent memory <id>            # ver memória e stats
mc agent skill <id> <skill>    # ativar/desativar skill
mc agent close <id>             # fechar zambia
          

          
// Bulletin Board

          
mc board read                    # ler mensagens recentes
mc board read --n 20             # últimas 20 mensagens
mc board post --topic alert "msg"  # postar mensagem
mc board reply <msg-id> "reply"    # responder mensagem
mc board delete <msg-id>           # remover mensagem
          

          
// Tarefas

          
mc task list                     # listar todas as tasks
mc task list --column doing      # apenas tasks em progresso
mc task add "título"             # criar task no backlog
mc task move <id> review        # mover para coluna
mc task remove <id>             # remover task
mc task show <id>               # detalhes da task
          

          
// Hive (colmeia)

          
mc hive broadcast "mensagem"     # broadcast para todos os agentes
mc hive snapshot                 # snapshot do estado atual
mc hive brainstorm "questão"     # brainstorm coletivo
          

          
// Chat REPL Interativo

          
mc chat <id>   # abre REPL com o agente
  /sair         # encerrar REPL
  /limpar       # limpar histórico
  /status       # ver status da sessão
          

        


        
        

          
Conceitos-Chave

          
Glossário do sistema para entender e operar com precisão.

          

            
ConceitoDefinição

            
SoulYAML com persona, diretivas e tom de comunicação. Define a "personalidade" do agente. Ex: kilo-native.yaml

            
SkillYAML com bloco de prompt injetado no contexto do agente. Expande suas capacidades. Ex: code-review.yaml

            
Character ChartFicha de atributos do agente (STR/INT/WIS etc.). Pode crescer organicamente via PATCH /api/characters/:agent

            
HeartbeatPulso periódico do agente. Ausência por >2min → status zombie. Registrado em hive/heartbeats.jsonl

            
ZambiaSub-agente temporário criado para missão específica. Tem parent, auto_close, encerra após idle 5min

            
LineageÁrvore de parentesco: agentes nativos + zambias indentados sob seu criador

            
chatKeyChave da sessão de chat. Para nativos: nome do agente (ex: "code"). Para zambias: ID do agente

            
BusMessage bus assíncrono (bus/messages.jsonl). 9 tópicos. Bulletin Board é a UI do Bus

            
DispatchFire-and-forget: envia prompt + cria task no Kanban + resposta via WS. Sem chat interativo

            
Context injectionSistema que prefixa a primeira mensagem do agente com soul+skills+character. Transparente ao Kilo

          

          
// Ciclo de Vida de uma Sessão

          
Ativar agente → buildAgentContext (soul+skills+char)
  → primeira mensagem = systemCtx + userMessage → Kilo
  → respostas subsequentes: apenas userMessage → Kilo
  → sessão expirada: recriar + re-injetar systemCtx automaticamente
          

        


        
        

          
Cooperação com IA

          
Como trabalhar com agentes como a Argenta como copiloto.

          

            Você não precisa operar o Mission Control sozinho. Agentes como a Argenta (orquestradora)
            podem co-pilotar o sistema — analisando estado, sugerindo ações, executando comandos.
          

          
// Padrão de Cooperação

          
    
            
  1. Você define a intenção em linguagem natural
    2. 
            
  2. A Argenta pesquisa e consolida — lê o estado, sugere uma rota
    3. 
            
  3. Você aprova ou ajusta — nunca assuma execução sem aprovação para ações irreversíveis
    4. 
            
  4. A Argenta executa e reporta — no Bulletin Board e no Kanban
    5. 
            
  5. Você valida o resultado
    6. 
          

          
// Comandos Úteis para a Argenta

          
"Analise o estado atual da colmeia e me dê um resumo"
"Crie um plano para implementar [feature X] e poste no Bulletin Board"
"Há agentes zombie? Tome as ações necessárias"
"Spawna um zambia para revisar este PR: [descrição]"
"Qual é o status das tasks em progresso?"
          

          
// Boas Práticas

          
    
            
  • Seja específico sobre o resultado esperado, não apenas a tarefa
    • 
            
  • Permita perguntas de clarificação antes de executar
    • 
            
  • Aprove explicitamente ações irreversíveis (deletar, fazer push, etc.)
    • 
            
  • Acompanhe o Bulletin Board para ver a coordenação entre agentes
    • 
          

          

            Dica: Abra o chat com ask (The Lore Keeper) para tirar dúvidas
            sobre qualquer conceito do sistema — ele conhece toda a documentação da colmeia.
          

          
// Injetando Contexto em um Agente IA Externo

          
O arquivo HELP-AI.md na raiz do projeto é otimizado para injeção como
            contexto de sistema. Basta colar seu conteúdo no prompt do agente e ele poderá
            operar o Mission Control via API sem precisar de contexto adicional.
          

        


        
        

          
FAQ

          
Dúvidas frequentes e soluções rápidas.


          
// O dashboard não abre

          
Verifique se o servidor está rodando com npm run ui.
            Se a porta 3030 estiver ocupada:

          
netstat -ano | grep ":3030 "
taskkill /PID <numero> /F


          
// O agente não responde

          
1. Verifique o status no Hive Panel — zombie = perdeu o heartbeat.

          
2. Verifique o Kilo: http://localhost:4096/global/health.

          
3. Abra o ACP e clique em "Reativar sessão".


          
// Como sei se o Kilo está funcionando?

          
Veja o indicador kilo-serve no header — verde = ok.
            Ou acesse http://localhost:4096/global/health diretamente.


          
// Posso usar qualquer modelo de IA?

          
Sim, desde que autenticado no Kilo. Os providers disponíveis aparecem nos seletores
            de provider/modelo no ACP e no Dispatch. Providers atuais:
            kilo · openai · github-copilot · kimi-for-coding · zai · bailian-coding-plan


          
// O que é um zambia e quando criar?

          
Zambia é um agente temporário com missão específica. Crie quando a tarefa é especializada
            demais para os 5 nativos, ou quando quer isolar um contexto (ex: "só para revisar este PR").
            Com auto_close, ele encerra sozinho ao ficar idle por 5 min.


          
// Onde ficam as conversas salvas?

          
    
            
  • ops/chat-sessions.json — sessões de chat (restauradas ao reiniciar)
    • 
            
  • bus/messages.jsonl — mensagens do Bulletin Board
    • 
            
  • hive/memory/*.md — memórias formais de agentes
    • 
            
  • hive/heartbeats.jsonl — histórico de heartbeats
    •
